Docker基本概念

Docker 映像

  • 组成应用程序运行所需的全部文件合集
  • 只读/读写 分层结构(每次对映像的修改都通过commit形成一个新层)
  • 容器层 是基于映像可读写的顶层
  • 映像层所有的Docker映像都源自于一个基础镜像(Debian/Ubuntu) –> 只读，不可修改i
  • 其他的功能模块附加于基础影像形成最终程序

  • Docker images 是构建Docker container的基础
  • Docker 层/容器层

每个映像有唯一的ID(SHA256/ 取前6字节来显示)

Docker容器
images是只读的，在基础上创建并运行容器(可读写)
